# OrbitDesk Workspace — Cron Job Setup

## cPanel Configuration

Log in to cPanel → Cron Jobs → Add New Cron Job

| Script | Schedule | Command |
| --- | --- | --- |
| Subscription Renewal & Auto-suspend | Daily 7:00 AM | `0 7 * * * php /home/USERNAME/public_html/shanfix/cron/check-subscriptions.php >> /home/USERNAME/logs/shanfix-cron.log 2>&1` |
| Trial Expiry Reminders | Daily 8:00 AM | `0 8 * * * php /home/USERNAME/public_html/shanfix/cron/trial-expiry.php >> /home/USERNAME/logs/shanfix-cron.log 2>&1` |
| Invoice Reminders | Daily 9:00 AM | `0 9 * * * php /home/USERNAME/public_html/shanfix/cron/invoice-reminders.php >> /home/USERNAME/logs/shanfix-cron.log 2>&1` |

Replace `USERNAME` with your cPanel username and adjust the path to match your installation.

```sql
-- Creates scheduled_email_log table used for deduplication
```

This table prevents the same reminder email from being sent more than once per subscription/invoice per billing period. The UNIQUE KEY `(event_type, reference_id, period_date)` ensures idempotency even if a cron fires multiple times in a day.


## Email Triggers

### Scheduled (Cron-based)

#### check-subscriptions.php

- **Renewal reminders**: Sends `Mailer::sendSubscriptionExpiry()` at 7, 3, and 1 day(s) before `ends_at` for `status='active'` subscriptions.
- **Auto-suspend**: Sets `status='expired'` and org `status='suspended'` for subscriptions where `ends_at < today`. Sends `Mailer::sendSuspension()` and an in-app danger notification.
- Reads SMTP config from `system_settings` via `getSettings()` (falls back to PHP constants).
- Deduplication: `scheduled_email_log` table — one email per (event_type, sub_id, ends_at_date).

#### trial-expiry.php

- Sends `Mailer::sendTrialExpiry()` at 7, 3, and 1 day(s) before `trial_ends_at` for `status='trial'` subscriptions.
- Also creates in-app warning/danger notifications for all org users.
- Deduplication: same `scheduled_email_log` table with `event_type` = `trial_reminder_{N}d`.

#### invoice-reminders.php

- **Due-soon**: Invoices with `status IN ('sent','pending')` and `due_date = today + 3 days` — sends once per invoice.
- **Overdue**: Invoices with `status IN ('sent','pending')` and `due_date < today` — sends once, then sets `status='overdue'` (natural dedup).
- M-Pesa paybill number loaded from `system_settings` (`mpesa_paybill`), not hardcoded.

### Event-Driven (Triggered in real time)

| Event | Trigger Location | Recipient | Channel |
| --- | --- | --- | --- |
| Invoice generated by admin | `admin/subscriptions.php` — `generate_renewal_invoice` | Client admin | In-app + Email (`sendInvoice`) |
| Admin replies to ticket | `admin/support.php` — `reply` (non-internal only) | Ticket creator (client user) | In-app + Email |
| Client opens new ticket | `client/support.php` — `create_ticket` | All super_admin users | Email |
| Client replies to ticket | `client/support.php` — `reply` | All super_admin users | Email |

## SMTP Configuration

SMTP is now read from `system_settings` (configurable in Admin → Settings → Email).
Constants in `config/database.php` serve as fallback if DB settings are empty:

```php
define('SMTP_HOST', 'mail.yourdomain.com');
define('SMTP_PORT', 587);
define('SMTP_USER', 'no-reply@yourdomain.com');
define('SMTP_PASS', 'your-smtp-password');
define('SMTP_ENC',  'tls');   // 'tls' | 'ssl' | 'none'
define('MAIL_FROM', 'no-reply@yourdomain.com');
define('MAIL_FROM_NAME', 'OrbitDesk');
```

If neither DB settings nor constants are defined the mailer falls back to `localhost:587` with PHP's built-in `mail()`.
